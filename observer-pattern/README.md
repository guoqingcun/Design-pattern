+ 定义

	观察者模式又叫发布/订阅者模式，定义对象间一对多的依赖关系，使得每当一个对象发生改变，依赖它的对象会自动得到通知。
	观察者中的四种角色：
	1. subject，被观察者
	2. concreteSubject,具体的被观察者
	3. observer，观察者
	4. concreteObserver具体的观察者
	
+ 优点

	1. 观察者与被观察者在抽象层耦合，符合依赖倒倒置原则。
	2. 两者都是单一职责胡实现，符合单一职责原则
	
+ 缺点

	1. 调式复杂
	2. 广播链长，不易控制
	
+ 场景

	1. 事件触发场景
	2. 系统间消息交换，消息队列场景
